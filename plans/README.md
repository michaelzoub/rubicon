## Findings considered and rejected (do not re-audit)

- **Session abort/SSE "IDOR" via sessionId** (`server.ts:1034-1060`): by design under the documented single-shared-key, public-x402 trust model; session ids are unguessable UUIDs. Revisit only if per-tenant keys arrive (noted in plans 004/011 maintenance).
- **Optional `RUBICON_AGENT_API_KEY` = open gateway**: by design — `docs/api-contract.md` states payment, not an API key, authorizes delivery. Only the non-constant-time comparison was actionable (plan 004).
- **`ws` 8.20.1 high-severity DoS advisory (GHSA-96hv-2xvq-fx4p)**: transitive via viem; the exploit needs a `ws` **server**, which this codebase never runs (HTTP/SSE only, Supabase realtime unused). Lockfile hygiene only — bump opportunistically when viem allows.
- **Supabase pooler `rejectUnauthorized:false` existence**: documented decision (commit 78d9601, `.env.example`). Only the silent override of explicit `sslmode=verify-full` was actionable (plan 004 step 3).
- **SSE 100-event replay cap** (`event-bus.ts:12`): kept; `/events` is live-tail + bounded replay, and plan 011's receipts endpoint is the authoritative post-hoc record. Documenting it further was folded into plans 008/011 notes.
- **`server.ts` god-module split (1220 lines)**: real debt, deliberately NOT planned now — it would collide with plans 005/006/009 editing the same handlers. Re-plan after those land.
- **`quickstart.ts`/`agent-client.ts` payment-loop duplication**: MED-confidence overlap; consolidation risks the highest-churn onboarding path. Reassess after the money-path chain stabilizes.
- **`polygres-test-app/` vendored experiment, `prd.json`, `skills-lock.json` at root**: repo clutter; removal is a one-line owner decision, not worth a plan file. Note `polygres-test-app/vendor/pgGraph/AGENTS.md` injects a second agent-instruction file into tools that scan the tree.
- **No request-body schema validation / no global error handler** (`server.ts` routes): real hardening, but M-effort with client-compat risk; deferred rather than rejected — promote to a plan when the money-path chain is done.
- **Per-word `recordWordDelivery` insert batching**: deferred until the contract suite covers batch idempotency (plan 009 maintenance note).
- **CLI formatting/args module tests** (`format.ts`, `args.ts`, `receipts.ts`): worthwhile, lower money-risk than what was planned; fold into a later test-coverage pass.
